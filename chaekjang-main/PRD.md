# PRD — 오늘의 한 칸 (독서 책장 앱)

> 문서 버전: v1.0 · 작성: WOWD.LAB · 스택: HTML/CSS/JS → Supabase

---

## 0. 한 줄 정의

**숏츠 1시간 대신 하루 20분, 독서를 "다시 펼치게" 만들고, 그 보상으로 나만의 책장이 채워지는 앱.**

---

## 1. 문제 정의 (왜 만드는가)

### 1-1. 진짜 문제는 "기록 부족"이 아니라 "시작 마찰"이다

사용자는 이미 독서를 원한다. 숏츠는 1시간이 순식간인데, 원하는 독서 20분은 어렵다.
이 비대칭의 원인은 *기록할 게 없어서*가 아니라 **책을 펴는 첫 행동의 마찰**이다.

> 핵심 판단: 기존 독서기록앱 대부분은 "읽은 뒤 기록"에 집중한다.
> 그건 이미 책을 한 번 든 다음의 일이다. 이 앱은 그 앞단,
> **"오늘 20분 펼치기"를 메인 무대에 세운다.**

### 1-2. 사용자가 진짜 원하는 것 (요청 원문 기반)

| 요청 | 표면 니즈 | 진짜 니즈 |
|------|-----------|-----------|
| 독서 습관을 지켜줘 | 알림·트래킹 | 시작 마찰 제거 |
| 올해 읽은 책 기록 | 목록 | 성취의 시각화 |
| 책 속 한 줄 저장 | 메모 | 다시 음미할 흔적 |
| 읽고 싶은 책 저장 | 위시리스트 | 다음에 펼칠 책 예약 |
| 표지로 책장처럼 | UI | 쌓이는 만족감 |

---

## 2. 타깃 사용자 & 핵심 시나리오

**주 사용자**: 독서 의지는 있으나 매일 시작이 어려운 직장인 (모바일 우선)

**핵심 시나리오 (Happy Path)**
1. 저녁, 리마인더 알림: "오늘의 20분, 아직이에요"
2. 앱을 연다 → 책장이 아니라 **20분 타이머**가 먼저 보인다
3. [시작] 한 번 누른다 → 부담 없이 읽는다
4. 20분 종료 → 진동 + 축하 → "오늘 읽은 책 고르고 한 줄"
5. 그 순간 책이 책장에 **꽂히는 모션** → 잔디(히트맵) 한 칸이 채워진다
6. 연속 기록(🔥)이 끊기지 않게 내일도 돌아온다

> 설계 원칙: **습관 → 한 줄 → 책장 → 잔디**가 하나의 흐름으로 묶인다.
> 기록은 습관의 *보상*이다. 따로 노는 기능이 아니다.

## 3. 기능 명세

### 3-1. v1 — 만든다 (이번 범위)

#### [F1] 20분 독서 타이머 ★핵심
- 앱 진입 시 **타이머가 첫 화면**. 책장보다 위.
- 원형 링 게이지 + 남은 시간. [시작]/[그만]/[분 설정].
- 종료 시: 진동 → 축하 모달 → 오늘 읽은 책 선택 → 한 줄 입력 → 책장 반영.
- **[F1-a] "딱 5분만" 비상 모드**: 20분이 부담인 날, 5분 미니 버튼.
  - 근거: 0분보다 5분이 낫다(행동설계). 마찰을 한 단계 더 낮춤.

#### [F2] 책장 (읽은 책) ★핵심
- 3열 그리드, 나무 선반 메타포(책등 그림자).
- 표지 자동(제목 검색) + 한 줄 미리보기 + 탭하면 상세.
- 신규 추가 시 "꽂히는" 드롭 애니메이션.

#### [F3] 책 속 한 줄
- 책별 인용 저장. 타이머 종료 흐름 또는 상세에서 입력.
- **[F3-a] 한 줄 모아보기 (필사 노트 뷰)**: 내 문장만 흐르는 별도 화면.
  - 근거: 다시 읽는 맛 = 재방문 동력.

#### [F4] 읽고 싶은 책 (위시리스트)
- 별도 탭. 표지+제목+저자. "읽기 시작" → 읽은 책으로 이동.
- 근거: 다음에 펼칠 책이 예약돼 있으면 시작이 쉬워진다.

#### [F5] 표지 검색
- 기본: 구글북스 API(키 불필요).
- 한국 책 정확도: 카카오 책 검색 API(REST 키 입력 시 우선 사용).
- 폴백: 표지 없을 때 제목 텍스트 커버 자동 생성.

#### [F6] 습관 유지 장치 ★핵심
- **[F6-a] 연속 기록(스트릭)**: 🔥 연속 일수 + 최근 7일 점.
- **[F6-b] 독서 잔디(히트맵)**: 1년 캘린더, 읽은 날 채워짐. 빈칸 채우고 싶게.
- **[F6-c] 저녁 리마인더**: 정한 시각에 "오늘의 20분" 알림.
  - 구현: PWA(웹 푸시/로컬 알림) 또는 알림 권한 기반.
- **[F6-d] 홈 화면 추가(PWA)**: 앱처럼 설치, 오프라인 기본 동작.

### 3-2. v2 — 미룬다 (이번 범위 아님 · 근거 명시)

| 기능 | v2로 미룬 이유 |
|------|----------------|
| 연간 목표(올해 N권) 게이지 | 숫자 압박이 *시작 마찰*을 키울 위험. 습관 안착 후 도입. |
| 통계 대시보드 | v1의 잔디·스트릭으로 충분. 과도하면 본질 흐림. |
| 별점·완독 상태 리본 | 만족도 기능. 핵심 흐름 안정 후. |
| SNS 공유 이미지 | 외부 동기. 내부 동기(습관) 먼저. |
| 친구 책장 비교 | 소셜은 별도 설계 난이도. v1은 1인 경험에 집중. |

> 교육 포인트: **"안 넣은 것"에 근거를 다는 게 PRD의 절반이다.**
> 무엇을 빼는지가 제품의 뾰족함을 만든다.

## 4. 비기능 요구사항

- **플랫폼**: 모바일 우선(폰), 데스크탑 호환. 최대 폭 480px 중앙 정렬.
- **디자인**: 종이 질감 + 나무 책장 메타포. 폰트 Pretendard(본문 14pt 기준).
- **성능**: 첫 화면 1초 내. 표지 이미지 lazy 로딩.
- **접근성**: 키보드 포커스, prefers-reduced-motion 존중, 터치 타깃 44px+.
- **오프라인**: PWA 캐시로 네트워크 없이도 타이머·책장 열람 가능.

## 5. 데이터 모델 (Supabase 기준)

> 상세 스키마·SQL은 `README.md`의 설정 섹션 참조.

- **books**: id, user_id, title, author, cover_url, shelf('read'|'wish'), year, rating(v2), created_at
- **quotes**: id, book_id, user_id, text, created_at
- **reading_logs**: id, user_id, date, minutes, created_at (잔디·스트릭 원천)
- **settings**: user_id, timer_minutes(기본 20), reminder_time, kakao_key

## 6. 성공 기준 (어떻게 "됐다"고 판단하나)

| 지표 | 목표 | 의미 |
|------|------|------|
| 7일 연속 사용률 | 측정 시작 | 습관 안착 |
| 타이머 완료율 | 시작 대비 70%+ | 20분 마찰 통과 |
| 책장 누적 권수 | 우상향 | 보상 루프 작동 |
| 한 줄 기록 비율 | 완료 중 50%+ | 흔적 남기기 정착 |

> v1 검증 질문: **"내일도 앱을 다시 여는가?"** — 이 하나로 충분.

## 7. 단계별 구현 순서 (바이브코딩 실습 트랙)

1. **Day 0 — 로컬 프로토타입**: 타이머+책장+한줄+위시(브라우저 저장). ← *현재 완료*
2. **Day 1 — 표지 검색 연결**: 구글북스 → 카카오 키 추가(품질 향상 시연).
3. **Day 2 — Supabase 연동**: 인증 + 4개 테이블 + RLS. *로컬→클라우드 전환이 교육 클라이맥스.*
4. **Day 3 — 습관 장치**: 잔디·스트릭·리마인더·PWA.
5. **Day 4 — 다듬기**: 한 줄 모아보기, 5분 모드, 마이크로 인터랙션.

> 핵심 메시지: **PRD가 있으면 바이브코딩은 "받아쓰기"가 된다.**
> 막막함은 PRD 부재에서 온다.
