# CLAUDE.md — 오늘의 한 칸 개발 지침

> 이 파일은 Claude Code가 이 프로젝트를 이어서 개발할 때 따르는 규칙이다.
> 작업 전 반드시 `PRD.md`와 `current_status.md`를 먼저 읽는다.

---

## 프로젝트 한 줄

숏츠 대신 하루 20분 독서를 "다시 펼치게" 만드는 모바일 우선 독서 책장 앱.
**시작 마찰 제거가 제1원칙.** 모든 기능 결정은 "이게 20분을 다시 펼치게 하는가?"로 검증한다.

---

## 기술 스택

- **프론트엔드**: HTML / CSS / JavaScript 정적 페이지 (기본)
  - React는 **사전 확인 후에만** 도입. 임의 전환 금지.
- **백엔드/DB**: Supabase (Postgres + Auth + RLS)
- **배포**: Vercel
- **표지 API**: 구글북스(키 불필요, 기본) / 카카오 책 검색(REST 키 입력 시 우선)
- **PWA**: manifest + service worker (오프라인·홈 추가·알림)

---

## 코딩 규칙 (필수 준수)

1. **한글 주석 필수.** 주요 함수·블록마다 한국어로 의도를 적는다.
2. **기존 코드 생략·삭제 없이 이어서 작성.** "// ... 기존 코드 ..." 식 생략 금지.
   전체 파일을 온전히 유지한 채 수정한다.
3. **폰트는 Pretendard.** 본문 14pt 기준, 최소 10pt.
4. **모바일 우선.** 최대 폭 480px 중앙 정렬. 터치 타깃 44px 이상.
5. **디자인 토큰 유지.** 종이/나무 책장 메타포. 색상은 CSS 변수(`:root`)로만 관리.
   - 종이 `--paper:#EDE4D3`, 나무 `--wood:#6B4A2F`, 포인트 `--accent:#C0472B`,
     보조 `--accent-2:#3E6B5A`, 금박 `--gold:#C89B3C`.
6. **시그니처 보존.** "타이머 종료 → 책이 책장에 꽂히는 드롭 애니메이션"은
   이 앱의 정체성. 함부로 제거·약화하지 않는다.

---

## 보안 규칙

- API 키(카카오 등)·Supabase 키는 코드에 하드코딩 금지.
  로컬은 `.env`, 배포는 환경변수로 관리.
- Supabase는 **RLS 필수.** user_id 기준으로 본인 데이터만 접근.
- 위험 명령(`rm -rf`, 강제 푸시 등)은 실행 전 사용자 확인.

## 작업 흐름 (Manager · Worker · Evaluator)

복잡한 작업은 역할을 나눠 수행한다.
- **Manager**: PRD 기준으로 작업을 쪼개고 우선순위를 정한다.
- **Worker**: 한 번에 하나의 기능을 온전히 구현한다.
- **Evaluator**: 10점 만점 자기평가. 9.0 미만이면 개선 버전을 제시한다.

작업 단위가 끝나면 `current_status.md`를 다음 포맷으로 갱신한다:
`[완료된 작업] / [진행 중 작업] / [다음 단계] / [미결정 이슈]`

## 기능 우선순위 (PRD 3장과 동기화)

### v1 (현재 범위)
- [F1] 20분 타이머 + [F1-a] 5분 비상 모드
- [F2] 책장 (읽은 책)
- [F3] 책 속 한 줄 + [F3-a] 한 줄 모아보기
- [F4] 위시리스트
- [F5] 표지 검색 (구글북스 → 카카오)
- [F6] 습관 장치: 스트릭 / 잔디 히트맵 / 저녁 리마인더 / PWA

### v2 (지금 만들지 말 것)
연간 목표 게이지, 통계 대시보드, 별점·완독 리본, SNS 공유, 친구 비교.
→ 사용자가 명시 요청하기 전엔 손대지 않는다.

---

## 절대 하지 말 것

- 책장을 첫 화면으로 올리기 (타이머가 먼저여야 한다)
- 연간 목표 권수를 강제 노출 (숫자 압박 = 시작 마찰 증가)
- 요청 범위를 넘는 기능 임의 추가
- 마크다운 과잉(문서 산출 시 HWP/PPT/Notion 복붙 고려)
